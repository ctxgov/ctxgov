# CtxVault Trademark And Official Project Policy

## Purpose

The `CtxVault` name identifies the maintainer-controlled project and official
release surfaces.

This policy is intentionally lightweight. It is meant to prevent confusing
official-affiliation claims while keeping normal open-source use easy.

## Allowed Use

You may use the CtxVault name to:

- refer to the upstream project
- describe compatibility with the public Apache-2.0 core
- identify forks as forks
- discuss or review the project
- publish integrations that accurately state they are unofficial

Examples:

- "Works with CtxVault"
- "A fork of CtxVault"
- "An unofficial CtxVault integration"

## Not Allowed Without Maintainer Permission

Do not use the CtxVault name, logo, or release language in a way that implies:

- your build is an official CtxVault release
- your app, package, service, domain, community, or binary is maintained by the
  CtxVault project
- your fork is endorsed by the maintainers
- your integration has passed official compatibility review when it has not
- you control official project infrastructure

Use distinct names for modified distributions unless the maintainers have
explicitly approved the official naming.

## Official Surfaces

Official project surfaces are maintained only by the CtxVault maintainers. The
public core is Apache-2.0, but the license does not grant rights to use
maintainer-controlled branding for unofficial releases.

## Reporting Confusion

If you see a package, app, domain, or community that appears to impersonate the
project or confuse official affiliation, open an issue in the official public
repository with concrete links and screenshots where possible.
