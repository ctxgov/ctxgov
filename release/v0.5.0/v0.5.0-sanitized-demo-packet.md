Boundary phrase for publication review: no target repository writes and no provider/model execution.

This packet demonstrates the v0.5.0 release claim:

> Governed context projection for AI work.

## What It Shows

1. Reviewed evidence becomes candidate context.
2. Owner or policy decisions caveat, block, or withhold candidates.
3. A directory projection preview is generated from reviewed material only.
4. Receipts record source refs, hashes, selected/omitted/blocked material,
   target-profile dry-runs, quality checks, and rollback paths.
5. No target repository, harness file, runtime, adapter, provider/model, memory
   store, benchmark, hardware, or cost surface is mutated or claimed.

## Demo Script

Use the sanitized example, not live repo mutation:

1. Open `examples/v0.5.0-governed-context-projection/README.md`.
2. Show `projection-preview/caveated.md` to explain how useful context is kept with
   explicit caveats.
3. Show `projection-preview/blocked.md` where applicable to show
   withheld context and unresolved questions.
4. Show `projection-preview/receipt.json` to
   prove file hashes, source refs, route decisions, and no-write boundaries.
5. Restate that public aggregate evidence is not benchmark, runtime, adapter,
   provider/model, hardware, or cost evidence.

## Public-Safe Wording

Allowed:

- CtxVault turns reviewed evidence, decisions, caveats, and receipts into
  portable context packets for AI tools, agents, and coding workflows.
- The v0.5.0 proof scene includes CtxVault dogfood plus owner-selected local
  OSS dry-runs.
- The dry-runs perform no hidden scan, provider/model call, adapter execution,
  runtime execution, memory promotion, or target writes.

Do not say:

- benchmark, leaderboard, reliability, or accuracy improvement
- adapter, provider/model, runtime, or hardware/cost compatibility
- automatic repository optimization
- stable Memory Governance Protocol
- Memory OS, RAG replacement, anti-hallucination prevention, or security
  certification
