# CtxVault v0.5.0 Public Demo Script Draft

Status: v0.5.0 public release artifact.

Release claim under review: Governed context projection for AI work.

Goal: show `Governed context projection for AI work` in three to five minutes
without publishing private material, mutating target repositories, or implying
benchmark/runtime/provider claims.

Boundary phrase for publication review: no target repository writes and no provider/model execution.

## Opening

CtxVault is a local trust layer for AI work. It does not ask an agent to scan
everything or silently write into another project. It turns reviewed evidence,
decisions, caveats, and receipts into portable context packets that an AI tool
or agent can consume with the boundaries still visible.

## Closing

The demo shows the core v0.5.0 capability: CtxVault can produce governed,
auditable context projection material while keeping uncertainty, caveats,
blocked content, receipts, and rollback paths visible.

It does not show benchmark superiority, runtime integration, provider/model
compatibility, hardware/cost improvement, automatic optimization, stable MGP,
Memory OS behavior, RAG replacement, hallucination prevention, or security
certification.

## Backup Line

If asked what makes this different from a context dump: every projected item is
bound to review decisions and receipts, and the withheld or caveated material is
part of the evidence instead of being swept away.
