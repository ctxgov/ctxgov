# CtxVault v0.5.0 Public Evidence Draft

Status: v0.5.0 public release artifact.

Release claim under review:

> Governed context projection for AI work.

This page summarizes the public-safe evidence behind the v0.5.0 claim. It uses
aggregate counts, repo-relative artifact refs, and explicit
claim boundaries. It does not include unpublished local paths, raw private
receipts, private source excerpts, provider/model outputs, target repository
writes, or specific local dry-run repository identifiers.

## What The Evidence Supports

CtxVault can turn reviewed evidence, caveats, blocked decisions, source refs,
hashes, manifests, and rollback instructions into portable context projection
material for AI tools, agents, and coding workflows.

The current proof scene covers one private dogfood run and three owner-selected
read-only OSS dry-runs. The OSS data below is deterministic local evidence only.
It is not a benchmark, leaderboard, reliability result, runtime integration, or
automatic repository optimization claim.

## Shared Boundaries

All three OSS runs keep these boundaries false:

- no hidden scan
- no target repository writes
- no harness writes
- no adapter execution
- no runtime execution
- no provider/model execution
- no memory promotion
- no public benchmark or leaderboard result
- no public reliability or accuracy result
- no hardware or cost claim

One run used a benchmark-shaped repository as ordinary repository material
only. No benchmark harness was executed and no leaderboard, reliability, or
automatic optimization claim is made from it.
