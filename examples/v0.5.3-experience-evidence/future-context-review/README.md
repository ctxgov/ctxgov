# Future Context Review Sanitized Example

Status: v0.5.3 public sanitized example.

Governed context projection for AI work.

Boundary phrase for publication review: no target repository writes and no provider/model execution.

This example shows a synthetic Future Context Review queue. It demonstrates
risk-ranked review ordering without writing owner decisions, projections,
target files, memory, public copy, provider/model outputs, or benchmark claims.

## Files

- `future-context-review-queue.json`: sanitized queue snapshot.
- `manifest.json`: source template, omitted refs, boundary, publication status,
  and rollback.

## What It Shows

1. Queue order is a review aid, not approval.
2. Each item keeps source support, recommended action, and rollback visible.
3. Weak source support and status-truth drift sort before lower-risk cards.
4. Side-effect booleans remain false.
