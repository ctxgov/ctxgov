# CtxVault v0.3.4 Release Notes

Status: v0.3.4 deterministic context extraction and quality-receipt milestone.

## Positioning

v0.3.4 makes the first public trial shorter: bring a static local source,
extract it, inspect the receipt chain, and verify that the projected context is
gated by deterministic privacy, budget, and quality checks.

The release keeps the baseline local and deterministic: no model call, no
embedding service, no vector database, no remote provider, no official live
connector, and no public Workbench or macOS app promise.

## Added

- `context-extract` CLI command for local source import, source fingerprinting,
  idempotency keys, slice rebuild, context prepare, and optional gated
  projection
- `context-extract --dry-run` for previewing planned imports without writing
  governed objects
- MCP `context.extract` over the same deterministic local behavior
- `receipt-inspect --latest --summary` for human-readable receipt-chain
  inspection
- MCP `receipt.inspect` with `summary_text`
- `ctxvault.context-extract-receipt/v1`
- deterministic context-quality, context-density, retrieval-gain,
  search-decision, source-conflict, source-retention, and prompt-patch-density
  checks
- read-only `doctor` checks for stale extraction source fingerprints, missing
  selection receipts, projection receipts whose context selection is gone, and
  blocked extraction runs
- static source-shape fixtures for ChatGPT, Claude, Gemini/messages, DeepSeek,
  Ollama/Open WebUI-style exports, Markdown vaults, and prompt files
- `scripts/run_v034_context_extract_stability.py`
- `scripts/run_v034_context_quality_scorecards.py`

## Preserved

- durable truth stays in governed local objects
- raw transcripts and static exports remain source evidence, not automatic
  durable memory
- projection remains gated by `handoff_ready`, privacy preflight, and token
  budget checks
- quality receipts are deterministic evidence surfaces, not model judgments
- v0.3.3 public review pack and v0.3.2 context-selection composer remain
  available

## Explicit Non-Goals

- no public Workbench or macOS app surface claim
- no stable public MCP compatibility promise for experimental context tools
- no model, embedding, vector, local model, remote provider, or LLM API
  execution path
- no official Obsidian, Cursor, Claude Code, Codex, or chat live connector
  claim
- no automatic durable-memory promotion from extracted source material
- no public token-saving, recall, accuracy, or agent-quality uplift claim
