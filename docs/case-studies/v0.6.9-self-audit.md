# CtxGov v0.6.9 Self-Audit: Post-Publication Context Drift

## Scope

This is a self-audit of public CtxGov repository text after the v0.6.9 public
evidence release. It does not claim external adoption, benchmark performance,
security coverage, provider/model compatibility, package publication, hosted
runtime, live adapter support, or CLI beta status.

## Context

CtxGov v0.6.9 was published as a public-safe Memory X-Ray evidence preview.
v0.6.10 then added the ASCR sibling-repo relationship. During the v0.6.11
public-surface hardening pass, several stale public-surface strings still
pointed at older publication or package state.

## Findings

### 1. publication_state_drift

Evidence:

- `release/v0.6.9/memory-xray-public-evidence-preview/README.md` still used
  pre-publication wording after the public release had already happened.
- The v0.6.9 GitHub release and publication receipts already recorded
  owner-approved publication actions.

Interpretation:

This is stale AI-facing release context. A future agent or reviewer could
incorrectly treat the public evidence release as incomplete.

Correction:

The evidence pack status now says it was published through the owner-approved
v0.6.9 public GitHub release while preserving provenance for the original
prepared state.

### 2. repo_map_drift

Evidence:

- `README.md` mapped the public package path to the legacy private namespace.
- The public repository now exposes `src/ctxgov/`, with legacy namespace
  references retained only as provenance in historical artifacts.

Interpretation:

This is path-level stale context. It can mislead agents, contributors, and
reviewers who use README text as operational context.

Correction:

The README repo map now uses `src/ctxgov/`, `schemas/`, public-safe fixtures,
and a separate companion repo reference.

### 3. version_surface_drift

Evidence:

- `pyproject.toml` still used `0.6.2.post1` and linked the changelog to
  v0.6.2.
- The public source release surface had moved through v0.6.9 and v0.6.10.

Interpretation:

Package metadata can become misleading public context even when package
publication remains blocked.

Correction:

`pyproject.toml` now aligns source metadata to v0.6.11 and links the changelog
to the v0.6.11 release. This is source metadata only and not a package
publication claim.

### 4. roadmap_pointer_drift

Evidence:

- `ROADMAP.md` still directed current public readers to v0.6.2 release files.

Interpretation:

This stale pointer could move reviewers away from the current public evidence
surface and toward older release boundaries.

Correction:

`ROADMAP.md` now points to the current public release surface: v0.6.11
hardening, v0.6.10 ASCR alignment, and v0.6.9 Memory X-Ray evidence preview.

## Boundary

This self-audit is a public-safe report-shape case study. It is not a public
benchmark result, security guarantee, external adoption claim, provider/model
compatibility claim, package publication claim, hosted runtime claim, live
adapter claim, stable ASCR standard claim, or CtxGov CLI beta claim.

## Reproduction

```bash
python3 scripts/check_public_surface_hardening.py
python3 scripts/check_public_evidence_release_pack.py
python3 scripts/check_ascr_aligned_release_pack.py
```

The checks are local and deterministic. They do not call providers, access
memory backends, publish packages, write external targets, or perform outreach.
