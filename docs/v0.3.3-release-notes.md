# CtxVault v0.3.3 Release Notes

Status: v0.3.3 safe context handoff hardening milestone.

## Positioning

v0.3.3 makes the deterministic context-selection loop easier for a reviewer or
operator to run without private project context. It is a public-package
hardening release over v0.3.2, not a scope expansion.

The release keeps the baseline local and deterministic: no model call, no
embedding service, no vector database, no remote provider, no official live
connector, and no public Workbench or macOS app promise.

## Added

- owner-operated public review pack built from a reusable scenario manifest,
  short public-source fixture summaries, boundary checks, and a synthetic
  blocked-selection fixture
- `scripts/run_v033_public_review_pack.py`
- human-readable owner reports at
  `artifacts/v0.3.3-public-review/owner-review.md` and
  `artifacts/v0.3.3-public-review/owner-review.html`
- public review summary at
  `artifacts/v0.3.3-public-review/summary.json`
- projection artifact and projection receipt for the first passing review
  scenario
- deterministic test coverage for ready, empty, over-budget, blocked-selection,
  and projection-receipt public review behavior
- public README, changelog, and package metadata aligned to v0.3.3

## Preserved

- v0.3.2 source-grouped context selection composer
- selected-slice privacy preflight before projection
- projection receipts linked to selected context
- local, file-backed deterministic correctness baseline
- experimental labeling for context selection, MCP, compiled state, doctor,
  projection receipt, healthcheck, and runtime receipt contracts

## Explicit Non-Goals

- no public Workbench or macOS app surface claim
- no stable public MCP `context.prepare` compatibility promise
- no write-capable MCP projection tool
- no model, embedding, vector, local model, remote provider, or LLM API
  execution path
- no official Obsidian, Cursor, Claude Code, Codex, or chat live connector
  claim
- no hosted user-data import or hosted product demo claim
