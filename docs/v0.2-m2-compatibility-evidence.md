# CtxVault v0.2 Compatibility Evidence

This document records the public evidence shape for v0.2 developer-framework
compatibility checks.

The evidence is experimental. It proves the inspectable local path, not stable
third-party compatibility guarantees.

## Deterministic Checks

Run:

```bash
python3 scripts/run_deterministic_checks.py
```

The extracted public core should pass syntax checks, fixture validation, and the
deterministic unit test suite.

## Projection Healthchecks

After generating projection outputs, run:

```bash
PYTHONPATH=src python3 -m ctxgov.cli adapter-healthcheck --root /tmp/ctxvault-clean-verify --target-kind agents-md --target-path exports/AGENTS.md
PYTHONPATH=src python3 -m ctxgov.cli adapter-healthcheck --root /tmp/ctxvault-clean-verify --target-kind claude-md --target-path exports/CLAUDE.md
PYTHONPATH=src python3 -m ctxgov.cli adapter-healthcheck --root /tmp/ctxvault-clean-verify --target-kind workstream-brief --target-path exports/workstream.md
```

Expected evidence shape:

- `schema_id` is `ctxvault.projection-healthcheck/v1`
- `status` is `pass` when target preflight succeeds
- `write_plan[*].will_write` is `false`
- `runtime_event_receipt.schema_id` is
  `ctxvault.runtime-event-receipt/v1`
- `policy_checks[*].decision` remains inspectable

## Optional Local Backup Evidence

Run:

```bash
PYTHONPATH=src python3 -m ctxgov.cli local-backup-write --root /tmp/ctxvault-clean-verify --target file:///tmp/ctxvault-clean-verify-backup --label "local backup rehearsal" --device-id local-target
PYTHONPATH=src python3 -m ctxgov.cli replica-verify --root /tmp/ctxvault-clean-verify --replica-root /tmp/ctxvault-clean-verify-backup --snapshot-id SNAPSHOT_ID_FROM_RECEIPT
```

Expected evidence shape:

- local backup write receipt schema is
  `ctxvault.local-backup-write-receipt/v1`
- replica verification schema is `ctxvault.replica-verify/v1`
- copied artifacts include a snapshot manifest, restore bundle, and sync
  manifest
- `replica_verification_status` is `verified`

Temporary local targets are acceptable for testing the product path, but public
copy should not claim external disk or NAS validation unless such a mounted
target has actually passed the same flow.
