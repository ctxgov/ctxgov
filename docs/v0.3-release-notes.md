# CtxVault v0.3 Release Notes

Status: v0.3.0 compiled context projection milestone.

## Positioning

v0.3 makes the reviewed source-to-context-to-projection loop more useful for
real AI work. It compiles current workstream state from governed local objects
and projects that state into practical work surfaces with receipts.

## Added

- experimental compiled workstream state read model
- `compiled-workstream-state` CLI command
- `workstream.compiled-state` MCP tool
- compiled current truth sections in `AGENTS.md`, `CLAUDE.md`, and workstream
  markdown projections
- read-only `doctor` CLI command
- `doctor.report` MCP tool
- `markdown-vault-import` CLI bridge for Markdown notes
- `.markdown` support for knowledge import

## Preserved

- canonical truth stays in governed local objects
- projection outputs remain downstream artifacts
- density warnings are advisory
- model adapters are outside the v0.3 baseline
- optional product surfaces remain outside the v0.3 baseline

## Experimental

- compiled workstream state read model
- doctor report
- `Episode`, `Workstream`, and `intelligence.py`
- plugin manifest, projection receipt, projection healthcheck, and runtime
  event receipt contracts

## Not Included

- official Obsidian plugin
- local or remote model adapters
- hybrid search or typed graph product promises
- optional product-surface availability claim
