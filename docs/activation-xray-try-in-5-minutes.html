Activation X-Ray Try-in-5-Minutes

Inspect what memory state was allowed to influence.

Run a local CtxGov demo that checks whether memory/context influence cases include source refs, authority ceilings, final-state assertions, rollback paths, and side-effect boundaries.

cd ctxgov

What You Can Test

  • Local single-user memory influence with source and rollback checks.
  • Team project memory boundaries with author and RBAC/ACL fields.
  • Cluster / multi-tenant memory-state surfaces with tenant, event, revision, tool, state, and audit fields.
  • Accepted and rejected cases compiled to MemoryOperationTrace, ConsequenceShard, PolicyGrant, and StateTransitionReceipt.

Release-Candidate Gate

To check the demo, public page, publish pack, output files, and claim boundary together, run the final local preflight python3 scripts/check_activation_xray_final_preflight.py. It includes focused unit tests, py_compile, and git diff --check. In the full source checkout it also runs docs checks and HN skip-live preflight; public slim checkouts report those absent full-source checks as skipped. The release-candidate gate is python3 scripts/check_activation_xray_next_milestone_release_candidate.py. The consolidated owner go/no-go gate is python3 scripts/check_activation_xray_publication_go_no_go.py. Social copy readiness is checked by python3 scripts/check_activation_xray_social_copy_ready.py. Local repeatability evidence is checked by python3 scripts/benchmark_activation_xray_scale_profile_demo.py; the runtime smoke evidence is local-only and does not create a public benchmark claim. Structured HN/X/LinkedIn payloads are rendered by python3 scripts/render_activation_xray_social_post_payload.py. The owner publish packet is rendered by python3 scripts/render_activation_xray_owner_publish_packet.py. The dry-run public patch publisher is python3 scripts/publish_activation_xray_public_patch.py --checkout <clean-ctxgov-checkout>. The gate also writes a minimal public publication bundle with file digests under release/activation-xray-scale-profile-demo/2026-06-10/. To apply it into a clean local public checkout without commit or push, run python3 scripts/materialize_activation_xray_publication_bundle.py --checkout <clean-ctxgov-checkout>. The exact publish commands are rendered by python3 scripts/render_activation_xray_publish_command_envelope.py, and the post-publish Pages check is python3 scripts/check_activation_xray_live_publication.py --live.

Claim Boundary

Unsupported public claims remain blocked. This demo creates no public savings number, benchmark result, compatibility or support claim, adoption claim, endorsement, security guarantee, or stable-protocol claim.

No provider/model call, runtime adapter, memory/backend write, target write, package publication, or outreach is performed by the local demo.
