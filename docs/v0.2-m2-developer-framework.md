# CtxVault v0.2 Developer Framework

v0.2 is a developer-framework milestone for the public `ctxvault` core.

It does not claim stable v1 status. It makes the local context core easier to
inspect and integrate with real AI work surfaces while preserving the
deterministic trust floor from M1.

## What v0.2 Adds

- read-only projection adapter healthchecks
- runtime event receipts attached to healthcheck output
- `AGENTS.md`, `CLAUDE.md`, and workstream brief target coverage
- optional local backup writes from governed snapshots to explicit local file
  targets
- CLI and MCP access to the same local core

## Projection Healthchecks

Projection healthchecks inspect whether a target surface is known, writable, and
compatible with reviewed-context projection rules. They do not write the target
file. They produce experimental evidence that an adapter builder can inspect
before relying on a target surface.

Supported public target kinds in this milestone:

- `agents-md`
- `claude-md`
- `workstream-brief`

## Runtime Receipts

Healthcheck output includes a runtime event receipt so users can inspect:

- actor
- event kind
- input refs
- selected target surface
- policy decision
- verification command

Runtime receipts are experimental developer evidence. They are not a model
correctness baseline and do not replace reviewed context, policy gates, or
projection receipts.

## Optional Local Backup Writes

`local-backup-write` creates a governed snapshot, emits a sync manifest, copies
the snapshot manifest and restore bundle into an explicit local file target,
and verifies that target as a replica.

This is optional local durability. It is not hosted sync, not a cloud backup
service, and not a public operator-backup contract.

## Non-Goals

- stable v1 language
- automatic memory promotion
- live connector promises
- hosted backup or sync guarantees
- making any single AI tool or coding harness the product center
