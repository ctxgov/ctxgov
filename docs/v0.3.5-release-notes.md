# CtxVault v0.3.5 Release Notes

Status: v0.3.5 first-run UX boundary patch over the v0.3.4 deterministic
context extraction and quality-receipt milestone.

## Positioning

v0.3.5 records the boundary for the maintainer-operated Workbench UX patch:
CtxVault may make first-run extract and inject easier in an optional product
surface, but the public core remains the deterministic CLI/MCP source-to-
context-to-receipt path.

The public release keeps the baseline local and deterministic: no model call,
no embedding service, no vector database, no remote provider, no official live
connector, and no public Workbench or macOS app promise.

## Added

- public release notes documenting the v0.3.5 Workbench UX boundary
- package metadata updated to v0.3.5
- explicit public wording that target profiles and Workbench convenience must
  not be read as runtime takeover, session attach, or agent-session management
- release-process guidance that demo data must be explicit local fixture
  seeding, not automatic private-data import

## Preserved

- public core extraction excludes Workbench and web assets
- v0.3.4 `context-extract`, `receipt-inspect`, doctor checks, and deterministic
  quality/stability scorecards remain the public trial path
- projection remains gated by `handoff_ready`, privacy preflight, token budget,
  and deterministic quality checks
- durable truth stays in governed local objects
- raw transcripts and static exports remain source evidence, not automatic
  durable memory

## Explicit Non-Goals

- no public Workbench or macOS app surface claim
- no runtime takeover for Codex, Claude Code, Cursor, terminals, or chat apps
- no automatic Git worktree creation, merge, or cleanup
- no mobile or remote-control handoff
- no LLM token streaming or provider-backed verifier execution
- no MemoryDB, vector, graph, or hybrid retrieval durable-truth integration
