# CtxVault v0.3.2 Release Notes

Status: v0.3.2 deterministic injection composer milestone.

## Positioning

v0.3.2 is a narrow fast-follow release after v0.3.1. v0.3.1 made local context
safe to slice, search, preflight, project, and purge as derived data. v0.3.2
adds the deterministic composition step between search and projection: users
can review source-grouped candidates, select exact slices, preview token budget,
run target-aware privacy preflight, and keep receipts that link selection to
projection.

The release remains model-free at the deterministic baseline. It does not add a
vector search baseline, model reranker, remote provider, official Obsidian
plugin, live connector, or graph truth claim.

## Added

- experimental `ctxvault.context-selection-receipt/v1` schema and fixture
- source-grouped local context selection composer
- explicit multi-slice selection over existing `ctxvault.context-slice/v1`
  refs
- token budget preview with `empty`, `within_budget`, and `over_budget`
  states
- target-aware privacy preflight embedded into composition results
- projection receipts linked to context selection refs and selection ids
- local pin, hide, archive, and clear preferences for slice suggestions
- `context-selection-compose` CLI command
- `context-slice-preference-set` CLI command
- `context-slice-preference-list` CLI command
- MCP tools for context selection compose and slice preferences
- deterministic tests covering composer receipts, preference behavior, CLI,
  MCP, projection receipt linkage, and open-core extraction

## Preserved

- canonical truth stays in governed source objects
- context slices remain rebuildable read models
- privacy preflight still blocks unsafe selected context before projection
- projection receipts remain downstream evidence, not canonical truth
- Markdown-vault import remains a bridge, not canonical storage
- baseline correctness has no model, embedding, vector database, remote
  service, or hosted API dependency

## Explicit Non-Goals

- no public workbench product claim
- no automatic source-material clearing
- no SSD, APFS snapshot, backup, cache, or filesystem secure-wipe claim
- no vector search baseline
- no graph product claim or canonical graph truth
- no model adapter baseline
- no silent remote reranking, embedding, enrichment, or external-send behavior
- no official Obsidian plugin
- no Codex, Claude Code, Cursor, shell, or AI-chat live connector claim
